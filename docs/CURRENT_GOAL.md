# 当前进行中目标

更新时间：2026-07-24

本文件是本项目当前唯一执行基线。后续旧方案、旧截图和旧文档若与本文件冲突，以本文件为准。

## 最终目标

在不改变 Codex 原生信息架构、组件尺寸、文本内容和交互热区的前提下，实现一套与《黑神话：悟空》深度融合的 Codex 视觉主题：

- 保持 Codex 原生页面结构，不增加额外侧边栏、底栏、主题控制栏或开关按钮。
- 正式包放入约定位置即可使用，并随 Codex 启动和退出；移除主题包后完整回归原生。
- 主题不得修改 `ChatGPT.exe`、`app.asar` 或官方配置，不得以长期 PowerShell 启动脚本作为最终方案。
- 关闭或移除主题后不得残留背景、标记、宠物、监听端口、watcher 或调试进程。
- 所有实现优先保证稳定与最小资源占用，不能以视觉效果换取崩溃风险。

## 当前实施顺序

### 1. 资源与稳定性硬门槛

这是当前第一阻断项，完成前不叠加宠物或复杂动效。

- 背景按需加载，不允许启动时预载全部场景。
- 稳态仅允许一张背景驻留；过渡期间最多两张。
- 不使用永久全屏 `will-change`、全屏滤镜或无必要的缩放合成层。
- 不做跨模式或相邻场景预取。
- 每次调试前后、每个长阶段至少每 30 分钟、任务收尾前检查一次进程、端口和内存。
- 只关闭本项目明确创建的调试实例、watcher、测试进程和工具进程；保留 Codex 主界面、renderer、GPU 与 app-server。
- 常态不保留主题调试窗口；实机截图与资源采样完成后立即关闭，并验证其 watcher、子进程和专用端口全部释放。

### 2. 背景与新建任务页

- 新建任务页使用战斗背景池；进入已有对话后切换为风景背景池。
- 两个背景池分别轮换，切换时使用低成本交叉淡化。
- 背景必须完整覆盖整个 Codex 窗口，不能只覆盖侧边栏或中央局部。
- 战斗视觉以杨戬与大圣为主，夜叉王为重要补充；其他 Boss 适量使用。
- 夜叉套、神锋、金箍棒等元素必须来源准确，不使用失真的手绘替代。
- 风景模式使用用户认可的纯画面、场景与建筑素材，不采用普通战绩图或杂乱战斗截图。
- 杨戬主背景使用用户指定的白底水墨对决图。
- 新建任务页原生标题和 56×56 图案允许替换，但必须保留原生占位、尺寸与布局；当前文案“此去，欲破何局？”与“金箍棒＋三道墨尾”图案属于待实机审计版本，不视为最终验收。

### 3. 输入框

- 先提交多个可比较的设计方案，由用户选择后再进入正式运行时。
- 保持原生输入框尺寸、最大宽度、布局、按钮位置和热区，只替换形状、边界、材质与细节。
- 不修改用户提示词、占位文案或回答内容；助手回答保持无额外外框。
- 黑神话元素必须通过结构、材质和克制的器物细节融入，不能仅换色，也不能生硬贴图。
- 当前 V8 三个候选方案仅供选择，未获选择前不得写入正式运行时。

### 4. 葫芦

- 暂不实现。
- 只提供其融入原生按钮、状态提示或局部交互的设计提案。
- 不增加独立葫芦面板、悬浮栏或装饰性占位。

### 5. Hatch Pet 宠物

两个宠物均使用 Hatch Pet v2 规范制作、修复、验证和打包。

- 名称固定为“小天命人”和“小八戒”。
- 小天命人使用高保真夜叉套与正确神锋；右脚夜叉护甲、武器前后完整性必须修复。
- 小天命人的跑动与棍花动作参考游戏动作；现有录屏只可用于背面跑动和棍花节奏，不推断不存在的正面细节。
- 小八戒保持正确九齿钉耙，外观参考游戏小八戒与 InArt 1/12 手办，整体更可爱但不失真。
- 小八戒悬停动作改为单手持钉耙、另一手捧腹大笑，动作节奏参考“奶龙捧腹大笑”，不直接复制角色造型。
- 宠物动作不得因悬停发生比例突变、装备变形、方向错误或不自然位移。

### 6. 最终随 Codex 启停

仅在背景、输入框和宠物全部完成并通过实机审查后集中处理。

- 优先寻找与 Codex 宿主生命周期绑定的原生或轻量宿主方案。
- 不把 PowerShell 常驻 watcher、启动快捷方式或手动重启作为正式交付。
- 保持“安装即用、移除即回原生”的用户体验。

## 不可违反的约束

- 不删除、移动或覆盖用户本地任何原始文件；只允许在项目或明确的主题安装目录中新增和修改本项目文件。
- 不使用 Computer Use 审查 Codex；以只读方式审计 `ChatGPT.exe` 所在应用包的 UI 源文件。
- 常态只保留当前 Codex 控制窗口；实机审查时最多临时增加一个主题调试窗口，截图与采样后立即关闭。
- 不新增表情符号装饰。
- 不改变 Codex 原生侧边栏宽度、输入框长度、环境信息窗口尺寸、按钮热区和回答排版。
- 每轮只运行与变更相关的定向测试；里程碑或推送前再运行必要的契约测试。
- 每轮维护需求、设计、分工与工作日志，并对本轮精确暂存、提交和推送；不得混入用户已有或无关改动。
