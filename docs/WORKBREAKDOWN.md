# 多对话分工与交付边界

本轮按用户要求启用三个并行子对话做只读审计，由主对话统一决策、修改、测试、安装与发布。子对话不直接写仓库，避免并行覆盖。

## 并行审计

| 对话 | 模型 / 推理 | 职责 | 对主实现的影响 |
| --- | --- | --- | --- |
| `asset_research` | `gpt-5.6-sol` / `xhigh` | 核对厌火套装、兽棍·神锋、金箍棒的正式名称、造型特征和网络参考 | 放弃通用甲片 SVG 和一像素“神锋”线，转为高保真透明 PNG；纠正“夜叉套”的正式套装身份 |
| `visual_architecture` | `gpt-5.6-sol` / `high` | 审计用户实机截图与 fixture 差异，定义侧栏、输入框和环境栏的非 AI 化视觉约束 | 收敛为“兵甲实物工作台”，去掉通用渐变卡片、金框、颗粒和装饰字形；环境栏只标记一个外卡 |
| `runtime_qa` | `gpt-5.6-terra` / `high` | 审计本机安装包、运行状态、DOM 标记器与测试盲区 | 定位了过宽外层 `form` 误标记、thread 判定过窄、助手外壳命中不足和安装包滞后问题；直接转化为回归测试 |

## 主对话责任

| 责任域 | 交付 | 证据 |
| --- | --- | --- |
| 产品边界 | 新建页战斗境、对话页风景境；无额外 UI、无文字改写、无尺寸修改 | `REQUIREMENTS.md` |
| 素材筛选 | 整理 11 张用户提供画面，用新的水墨杨戬替换旧杨戬图，统一压缩 | `themes/assets/` 和 `ASSET_SOURCES.md` |
| 高保真装备 | 基于装备参考编辑厌火套装、兽棍·神锋透明 PNG；被否决方案不进入运行包但历史内容保留 | `themes/motifs/` |
| DOM 适配 | 双境状态、稳定风景哈希、战斗场景循环、composer 尺寸筛选、助手祖先链透明化 | 动态 fixture |
| 视觉实现 | 侧栏厌火套装、输入框金箍棒 + 神锋、环境栏套装细节、多背景 | 原生基线 + 7 张定向实渲染图 |
| Windows 生命周期 | 固定根目录、版本化 release、同启停 watcher、renderer 停用与 append-only 保留 | 受管包 / 零删除合同 |
| 发布 | 精确 diff、只暂存本轮文件、commit、push | Git 本地与远端 SHA |

## 0.7.0 里程碑

1. 素材仓扩展为 11 张电影画面，按 battle-primary、battle-secondary 和 scenery 分组。
2. 用户指定的白场水墨杨戬图替换旧梅山杨戬背景。
3. 新建任务页和对话页按用户最后指令反转为“战斗 / 风景”。
4. 高保真厌火套装与兽棍·神锋真正进入运行时样式，金箍棒使用实景图；旧手绘 SVG 不再交付。
5. 修复输入框拉长、thread 判定、助手外壳与环境卡嵌套四类风险。
6. 直接读取 `app.asar` 的 UI 文件后，把 fixture 收敛到 275 px 侧栏、46 px 工具栏、736 px composer 和 300 px 浮动环境卡。
7. 定向回归、零删除本地版本化安装、生产窗口边界审计和单轮推送是本里程碑的完成条件。

## 交接规则

- fixture 截图不得称为真实 Codex 生产截图。
- 不修改 `app.asar`、WindowsApps、`ChatGPT.exe` 或官方快捷方式。
- 不强关 Codex；已打开普通 Codex 无端口时，明确报告无法原窗口热注入。
- 主对话及公共安装/停用入口不删除或移动任何文件；失败中间物与旧 release 原位保留。
- 不增加主题侧栏、底栏、状态卡或 in-app 开关。
- 不用整份 `config.toml` 备份覆盖用户后续设置。
- 每轮只运行覆盖当前变更风险的测试，并精确暂存路径。
