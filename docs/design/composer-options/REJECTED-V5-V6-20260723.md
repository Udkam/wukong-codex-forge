# Composer V5 / V6 rejection record — 2026-07-23

本文件只追加失败结论；V5、V6 的页面、截图、几何报告与源文件均原位保留，不删除、不覆盖。

## V5：native shape study

结论：失败，禁止进入 runtime。

- A 在 1× 下读成泛用的棱角科幻胶囊，黑神话语义依赖标题解释。
- B 读成带斜纹的通用面板，仍然是“给原生长框换皮”，没有形成游戏中的形状语法。
- 几何状态通过只能证明没有改动宿主尺寸与命中区，不能替代审美验收。

## V6：厌火襟匣 / 如意棍枕

结论：失败，禁止进入 runtime。

- `厌火襟匣` 的大号青色核心与放射齿形在无标题 1× 下更像泛用奇幻宝石徽章，无法被识别为兽棍·神锋；左侧衣襟也只剩抽象色块，不构成完整夜叉衣甲关系。
- `如意棍枕` 把整条输入器直接画成金红武器条，在无标题 1× 下更像科幻状态条；它虽然包含端箍和棍身，却没有解决“把道具贴到框上”的根本问题。
- 两案都通过 16 态 DOMRect、无外扩、命中区与 reduced-motion 验证，但视觉盲审失败，因此自动验证结果不得表述为方案通过。

## 冻结边界

- V5、V6 不接入 `runtime/`、正式包或本地 Codex。
- 不围绕两案继续做提亮、换色、缩放、锐化或图标替换。
- 不再要求同一个 composer 同时容纳夜叉套、兽棍·神锋和金箍棒。
- 夜叉套与神锋由 Hatch Pet 小天命人完整表现；金箍棒由战斗背景完整表现；composer 只承接《黑神话：悟空》原生界面的留白、墨迹、器物边缘与信息层级，避免道具堆砌。

下一候选仍须保持原生 `736×96` / `560×96`、全部原控件 DOMRect、提示词与命中区；在无标题 1× 下若只能读成“古风”“奇幻”或“游戏 HUD”，即直接失败。
