# 大圣归来 Codex 原生主题 — 需求与验收

## 当前目标

在不改变 Codex 页面结构、不新增控制栏、不启动第二实例的前提下，为 Windows Codex 应用一套可安装、可精确卸载的《黑神话：悟空》玄金主题。

“下载即用、删除即原生”在本项目中的可验证定义是：运行 `install-theme.cmd` 将正式包放入 Codex 用户目录并写入原生主题设置；运行 `remove-theme.cmd` 恢复安装前的受管设置并删除正式包。安装器不关闭或启动 Codex。

## 功能验收

| ID | 需求 | 验收标准 | 状态 |
| --- | --- | --- | --- |
| F-01 | 原生页面替换 | 页面结构、侧栏、环境栏、输入区与事件全部仍由 Codex 提供；主题不注入 DOM。 | 已实现 |
| F-02 | 无额外控制 UI | 不创建主题按钮、主题状态栏、侧栏、底栏或开始菜单入口。 | 已实现 |
| F-03 | 安装到对应位置 | 正式包只安装到 `%USERPROFILE%\.codex\themes\wukong-codex-forge`。 | 已实现 |
| F-04 | 原生玄金视觉 | 使用 Codex 支持的 accent、surface、ink、contrast、fonts 和 semanticColors。 | 已实现 |
| F-05 | 删除恢复 | 卸载只恢复仍由主题持有的值；用户后改值和所有无关配置保持不变。 | 已实现 |
| F-06 | 用户背景素材 | `大圣归来.jpg` 随包作为预览素材保留。 | 已实现 |
| F-07 | 新建/对话双背景 | 原生 schema 没有背景图与页面状态字段，不在 0.3.0 伪实现。 | 原生能力阻塞 |
| F-08 | 小悟空宠物 | 原生 schema 没有装饰节点插槽，不在 0.3.0 注入。 | 原生能力阻塞 |

## 稳定性与成本

| ID | 约束 | 验收标准 |
| --- | --- | --- |
| S-01 | 不修改官方程序 | WindowsApps、`app.asar`、签名文件和 Codex 可执行文件零写入。 |
| S-02 | 无接管进程 | 不使用 CDP、远程调试端口、watcher、独立用户 profile 或专用 ChatGPT 启动器。 |
| S-03 | 零页面冲突 | 不加载自定义 CSS，不增加 DOM，不覆盖哈希类名或宿主几何。 |
| S-04 | 小安装包 | 正式目录仅含主题定义、恢复状态、引擎、许可和 78 KB 预览图。 |
| S-05 | 受控删除 | 删除目标必须是固定主题目录，且通过 state marker、路径和 reparse point 检查。 |
| S-06 | 配置不丢失 | 卸载按键恢复，不用整份旧配置覆盖安装期间产生的新设置。 |
| S-07 | UTF-8 | 脚本与 JSON 使用 UTF-8；Windows PowerShell 入口保持 ASCII-safe。 |

## 发布与记录

- 每个可验证里程碑单独提交并推送，不积攒多轮未发布改动。
- 每轮运行与修改范围对应的测试；Studio E2E 仅在预览代码变化时运行。
- 需求、设计、分工在 `docs/` 版本化；本地过程日志写入 `docs/logs/CHANGELOG.md`。

## 验收边界

自动测试证明配置转换、恢复、安全路径和包结构；颜色是否符合主观审美由用户在当前 Codex 页面审计。背景和宠物不能以原生主题字段表达，不能将 Studio 截图当成实际 Codex 运行证据。
