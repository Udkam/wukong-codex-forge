# 大圣归来 Codex 主题 — 需求与验收

## 目标

在 Windows ChatGPT/Codex 桌面端提供一套与《黑神话：悟空》融合的本地主题。它应有明确的作品气质，而不是单纯换背景；同时把稳定性、可恢复性和运行成本放在视觉效果之前。

本轮默认主题名为“大圣归来 · 残卷入梦”。核心叙事是：

- 新建对话是“启程”：夕照、剪影和大标题承担视觉焦点。
- 对话进行中是“行路”：同一素材降到低显影，只留下余烬和轮廓，内容重新成为第一层级。
- 卸载主题是“归卷”：Codex 原生表面、布局和交互恢复，主题不继续覆盖原生样式。

## 功能验收

| ID | 需求 | 验收标准 |
| --- | --- | --- |
| F-01 | 安装即用、卸载即原生 | 运行时不创建主题按钮；安装后从受管 ChatGPT 入口启动即启用，卸载删除受管样式、节点和入口并恢复原生。 |
| F-02 | 新建对话独立设计 | 无消息时自动识别为 `landing`，在原生新建任务页面显示高显影背景；不注入标题卡或重排原生内容。 |
| F-03 | 对话态差异 | 出现消息后自动识别为 `thread`，背景显影显著降低，消息、代码和输入区获得稳定承载面。 |
| F-04 | 侧栏与按键 | 原生新建对话按钮、活动项目、普通按钮、图标按钮、菜单和弹窗使用烬金/黛玉主题反馈；不改变位置、尺寸、圆角、间距或事件。 |
| F-05 | 小行者 | 作为唯一可选装饰节点，可关闭、左右停驻、调整尺寸；不接受指针事件，不显示状态气泡。 |
| F-06 | 本地素材 | 默认使用用户提供的 `大圣归来.jpg`；受管导入允许 PNG/JPEG/WebP，单文件上限 16 MB。 |
| F-07 | 高可读性 | 高可读性预设移除图片、停止动效并提高暗化。 |
| F-08 | 配置导入导出 | 主题使用 schema v2 JSON；Node 运行时对主题和受管素材路径做校验。 |

## 稳定性与成本

| ID | 约束 | 实现边界 |
| --- | --- | --- |
| S-01 | 不修改官方程序 | 不修改 WindowsApps、`app.asar`、Codex 配置或任何签名文件。 |
| S-02 | 仅本机回环 | 调试端点必须是 `127.0.0.1`，并验证监听进程位于注册的 `OpenAI.Codex` 包内。 |
| S-03 | 随启随停 | 受管启动器启动 ChatGPT 和 watcher；ChatGPT 的 CDP 端点退出后 watcher 自动退出。 |
| S-04 | 低运行成本 | 仅一张 78 KB 背景、一个可选宠物；安装时不复制 Studio、测试、Git 数据或开发依赖；无远程字体、无视频、无 WebGL；DOM observer 采用 120 ms 合并刷新，watcher 1.4 s 轻量探测。 |
| S-05 | 可恢复 | restore 删除样式、受管节点、所有 Forge 标记、observer 与状态属性；卸载只允许精确受控目录。 |
| S-06 | DOM 变化容错 | 找不到某个语义节点时跳过该节点，不替换应用 markup，不阻断输入、点击或路由。 |
| S-07 | 原生结构不变 | 主题不得创建自定义侧栏、底栏、落地标题卡或控制按钮；除可选小行者外不向 body 添加主题节点。 |

## 发布与记录

- 每个可验证里程碑单独提交并推送，不积攒多轮未发布改动。
- 每轮只运行与改动相关的测试；全量测试留给发布里程碑。
- 需求、设计、分工在 `docs/` 下版本化；过程工作日志保存在本地 `docs/logs/CHANGELOG.md`，不进入连接仓库。

## 当前验收边界

浏览器 fixture 能证明配置、双视觉状态、原生几何不变、无额外栏位/按钮和清理契约；全窗口预览页只提供视觉证据，不显示主题编辑栏。真实 ChatGPT 版本的最终视觉验收必须从受管主题快捷方式重新启动应用；当前任务本身运行在未开启 CDP 的 ChatGPT 进程中，不能在不终止本任务的前提下完成该一步。
