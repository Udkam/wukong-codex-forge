# 大圣归来 Codex 样式层 — 需求与验收

## 当前目标

在不改变 Codex 原生三栏布局、不新增控制栏、不修改官方程序文件的前提下，实现一套明亮、可安装、可恢复的《黑神话：悟空》深度样式：背景、新对话页面、侧栏按钮、输入框、消息和环境卡必须出现新的主题语言，而不是只替换颜色 token。

“下载即用、删除即原生”的交付定义为：安装后使用受管 Codex 入口即可加载；卸载先恢复打开窗口和原生配置，再删除受管文件。当前已运行的普通 Codex 无调试入口，安装器不会强制重启或对其做进程注入。

## 稳定性与成本

| ID | 约束 | 验收标准 |
| --- | --- | --- |
| S-01 | 官方程序零写入 | 不修改 WindowsApps、`app.asar`、签名或 `ChatGPT.exe`。 |
| S-02 | 回环隔离 | 调试通道只绑定 `127.0.0.1`，随机端口，目标只接受 `app://codex` 或本地页面。 |
| S-03 | 最小运行包 | 不复制 `.git`、`docs`、Studio 或测试；只包含 runtime/shared/themes、启动脚本、许可和 `ws`。 |
| S-04 | 低轮询成本 | MutationObserver 110 ms 合并，watcher 1.7 s 廉价探测，不做连续截图或布局轮询。 |
| S-05 | 几何零侵入 | CSS 不覆盖宿主宽高、网格、固定槽位坐标；每轮逐槽比较 DOMRect。 |
| S-06 | 受控删除 | 固定路径、state marker、profile/shortcut 路径和 reparse point 多重检查。 |
| S-07 | 配置不丢失 | TOML 按键恢复；用户后改值和无关配置保持不变。 |
| S-08 | UTF-8/PS 5.1 | 中文文档/JSON UTF-8；PowerShell 生命周期脚本保持 ASCII-safe。 |

## 发布与记录

- 每个可验证设计里程碑独立 commit/push；网络失败必须明确保留为未完成项并重试。
- 每轮只运行覆盖当前风险的测试；不因安装器变化重跑 Studio 全量 E2E。
- 需求、设计、分工、运行调查在 `docs/` 版本化；过程日志写入本地 `docs/logs/CHANGELOG.md`。

## 证据边界

自动 fixture 截图证明 CSS、DOM 标记与几何契约，不等于真实 Codex 生产窗口验收。只有受管入口启动后的真实截图和用户审计，才能将 F-10 标记为完成。
