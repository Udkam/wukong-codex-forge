## 结论先行

《黑神话：悟空》的辨识度不来自“暗底 + 金色圆角边框”，而来自以下组合：

1. **单侧锚定的墨面**：内容区从一侧聚拢，另一侧溶入场景或留白，不形成完整卡片。
2. **一条很轻的断续细线**：只承担层级分隔，不围成四边框。
3. **一个器物或印章焦点**：卷轴端、铜钮、徽记、朱印只出现一次，不沿边缘重复铺装饰。
4. **有方向的断口**：木料崩口、飞白、墨蚀、卷纸翘边都有明确受力方向，不是随机噪点。
5. **大面积静默留白**：装饰聚在 20%–35% 的局部，其余空间让文字和交互保持清楚。

因此，736×96 Codex composer 不应继续做“发光金框”“全周武器栏”“左右对称护栏”或“缩小的装备照片”。最适合的签名动作是：

> **展开的经卷 + 单侧墨蚀断口 + 一枚铜铸发送钮**

它能让输入框一眼属于《黑神话：悟空》，同时仍保持 Codex 原生尺寸、文本宽度、点击区域和信息密度。

## 736×96 composer 结构规范

以下尺寸只描述视觉占位，不改变 Codex 原生命中框或文本区域。

### 1. 外壳

- 继续使用 Codex 原生 `736×96` 外接尺寸。
- 主体应接近“无完整边框”的暗墨/纸面，不拉长、不加高。
- 圆角不是主题特征：保留原生必要圆角时，将视觉边缘用卷纸、墨蚀或断线打断，避免看成普通 SaaS 卡片。
- 不给回答正文加框，不改变用户提示词和回答排版。

### 2. 左侧锚点

- 宽度：`56–72 px`。
- 形状：半展开卷纸、木案断面或方印三选一，不混成装备陈列。
- 仅允许一个主焦点；卷轴端头可略微突出，但不能侵占 textarea。
- 纹理只用低对比纤维、浅刻痕和一处不规则缺口。

### 3. 细线与断口

- 从 `x≈72` 延伸到 `x≈650` 的 `1 px` 细线即可。
- 线必须至少有一次 `36–64 px` 的断开，不能形成闭合框。
- 底部只保留 `180–240 px` 的短墨痕或纸缘，不做全宽金色 rail。
- 断口应朝一个方向展开，避免均匀锯齿或程序化噪声。

### 4. 右侧发送座

- 视觉占位：`36–40 px`，命中区继续使用原生尺寸。
- 轮廓来源：铜鼎莲座、金箍卷云、丹炉叠檐，三者只选一种主型。
- 默认态表现为暗铜浅浮雕；悬停和可发送态通过内层结构抬起、阴影分离或刻纹变清楚表达。
- 禁止把神锋、夜叉套或金箍棒整张缩略图塞进发送钮；高保真器物更适合作为背景/宠物或独立展示，不适合作为 40 px 微缩装饰。

### 5. 留白与信息

- 视觉装饰总占比控制在 composer 面积的 `20%–35%`。
- 至少 `60%` 区域保持安静，优先保证输入文字、工具入口、模型状态和发送状态可读。
- 印章、篆刻、暗纹、金属纹每个状态最多出现一种，避免“所有元素都要出现”的拼贴感。

## 推荐的状态语言

| 状态 | 非颜色变化 | 应避免 |
| --- | --- | --- |
| 默认 | 墨面平伏，细线若隐若现，铜钮内收 | 完整金框、常亮外发光 |
| 聚焦 | 卷纸边缘略抬，断线两端变清晰，铜钮浅浮雕分层 | 整框变亮、输入框高度变化 |
| 有内容 | 右侧铜钮外层旋出一小级，刻纹清晰 | 金箍棒/神锋照片突然出现 |
| 可发送 | 铜钮台阶抬起 1–2 px，阴影变窄 | 放大按钮、改变命中区 |
| 错误 | 一处短飞白/朱印式裂痕出现 | 全框红色描边、警示 emoji |
| 禁用 | 浮雕层压平，细线中断更长 | 透明度降到难以辨识 |

## 明确排除

- 普通战斗画面、角色正面拼贴、全屏 Boss 截图。
- 全周金色边框、对称武器护栏、连续庙檐牙口。
- `64×8` 或类似极窄服装切片冒充夜叉套高保真还原。
- 巨大题字覆盖输入区域。
- 多种法宝、套装、武器同时挤入 736×96。
- 玻璃拟态卡片、霓虹描边、通用“AI 生成古风”纹样。
- emoji、额外侧栏、额外底栏、改变 Codex 原生尺寸与布局。

## 下一轮实现建议

优先只做一个 composer 原型，不同时改动侧边栏、背景和宠物：

1. 先按“经卷 + 墨蚀 + 铜铸钮”实现 `736×96` 的默认、聚焦、有内容、可发送四态。
2. 用 1× 和 4× 两种截图检查：文字宽度、原生按钮位置、断口是否像真实材料、装饰是否超过 35%。
3. 通过后再把同一套形状语法迁移到侧边栏选中态和环境信息窗口；不是复制同一张纹理，而是复用“单侧锚点 + 断线 + 单焦点”。

本报告采用 frontend-design 的“一个签名动作”原则：以经卷展开为唯一主隐喻，其余墨蚀、铜铸、篆刻只承担层级和交互，不再堆叠成主题素材清单。
